### 已较接近主视觉

### 明显仍在分叉

- 导出体系的选择栏、进度卡片、局部按钮、toast

- 时间线 preview panel / status badge / context menu / toggle

## 统一样式的首批落点

建议按以下优先级落地：

1. 建立共享视觉 token：
   - `surface`
   - `control-bg`
   - `border`
   - `shadow`
   - `accent`
   - `text`
   - `muted`
   - `success / warning / error`
2. 用 token 接管内容侧总线：
   - `public/contentStyle.css`
   - `src/features/styles/timeline.css`
   - `src/features/styles/controlCapsule.css`
3. 清理内嵌 style 字符串：
   - `src/features/networkQuality/pagePanel.ts`
   - `src/features/watermarkRemover/statusToast.ts`
   - `src/features/quoteReply/index.ts`
4. 二次统一复合界面：
   - 文件夹体系
   - 导出体系
   - 时间线 preview / context / badge

## 风险说明

- `public/contentStyle.css` 是内容侧 UI 的中心总线，改动收益最大，但 blast radius 也最大。
- `src/pages/content/folder/manager.ts` 与 `src/pages/content/export/index.ts` 都是复合 UI 集散地，任何视觉变量命名不稳都会波及多个弹层。
- `src/features/networkQuality/pagePanel.ts` 和 `src/features/ui/controlCapsule.ts` 使用 Shadow DOM / 内嵌 style，必须单独同步 token，否则外层 CSS 不会自动覆盖。

## 本报告用途

- 作为“所有 UI 是否已纳入统一改造”的核对底表。
- 后续每完成一轮样式统一，都应回到本文件逐类核验覆盖情况。
