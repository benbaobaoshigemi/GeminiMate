# GeminiMate 进阶路线图 (Roadmap)

## 第一阶段：架构迁移 (已完成 ✅)
- **目标**：将实验性的 `GeminiHelper` 原型迁移至专业级框架。
- **成果**：
  - 基于 **React 19 + TypeScript 5.8 + Vite 7** 的全新架构。
  - 实现 **高级玻璃拟态 UI**，优化了交互布局与视觉识别度。
  - **Repair Engine**：整合了 LaTeX 深度修复与 Markdown 增强逻辑（水合、边界修复、粗体修剪）。
  - **性能提升**：通过 CRXJS 构建优化，确保对网页性能影响最小。

---

## 第二阶段：核心功能移植 (进行中 🏃)

我们将从参考项目中通过“外科手术”方式拆解并移植以下高价值功能：

### 1. 水印移除 (Watermark Remover) - **最高优先级**
- **方案**：劫持 `fetch` 响应或动态监听 DOM，切除 Gemini 注入的隐形水印。
- **状态**：待实现。

### 2. 公式复制增强 (Formula Copy)
- **方案**：移植 `FormulaCopyService`，支持三种格式点击复制：
  - 标准 LaTeX
  - **MathML (Word 完美兼容)**
  - 纯文本
- **状态**：已实现 ✅ (并集成 temml 生成 MathML)。

### 2.5 时间线 (Timeline) - **深度定制**
- **方案**：移植 `TimelineManager` 并根据用户需求深度二次开发：
  - **厚度调节**：废弃开关，改为 8px-32px 的滑块无级调节，通过 CSS 变量动态映射热区与节点大小。
  - **模式控制**：完整支持“平滑流动”与“瞬间跳转”双模式切换。
  - **防跳容器**：支持隐藏原始对话容器以消除长对话加载时的闪烁。
  - **存储同步**：统一采用 `chrome.storage.local` 解决 Popup 与页面脚本的通讯延迟。
- **状态**：已实现 ✅。

### 3. UI 极致调优 (Layout Polish)
- 背景毛玻璃效果。
- 侧边栏宽度可调节、输入框自动折叠。
- **Mermaid 表格直接渲染**。

---

## 第三阶段：核心杀手锏 - Word 一键导出 (Heavyweight)

这是我们接下来的技术攻坚点：
- **流程**：`Markdown 内容` -> `LaTeX 修复` -> `转为 MathML` -> `封装 WordML` -> `生成 .docx`。
- **目标**：解决目前所有 Gemini 插件导出 Word 时公式变乱码的痛点，输出出版级的 Word 文档。

---

## 验证与验收

1. **架构验证**：运行 `npm run build` 生成 `dist` 文件夹。
2. **UI 验收**：点击插件图标，确认呈现出 premium 风格的控制面板。
3. **修复验证**：开启开关，确认公式后的间距与粗体修复标记（虚线）正确呈现。
