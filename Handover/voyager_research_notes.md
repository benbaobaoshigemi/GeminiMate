## 二、架构水平评估

### 技术栈
| 维度 | 详情 |
|------|------|
| 框架 | Vite + TypeScript + React |
| 目标 | Chrome / Firefox / Safari (三套 vite config) |
| 国际化 | 10 种语言 (i18n) |
| 认证 | Chrome `identity` API + OAuth2 (Google Drive) |
| 测试 | Vitest + 单元测试 |
| CI/CD | GitHub Actions (31 个 workflow 文件) |
| 代码规范 | ESLint + Prettier + commitlint |

### 架构亮点
1. **渐进初始化**：区分前台/后台标签，后台3-8秒随机延迟防止"雷暴群"。
2. **特征解耦**：每个 Feature 都是独立模块，有 `start()` 和 `destroy()` 生命周期。
3. **平台感知**：根据 URL 自动区分 Gemini / AI Studio / Enterprise / 自定义网站。
4. **合并而非覆盖**：云同步的下载逻辑使用 `mergeFolderData` / `mergePrompts` 而非覆盖。

### 不成熟之处
1. **LaTeX 修复几乎为零**：`markdownPatcher` 只修了 bold 标签，`katexConfig` 只是注入 `strict: false`。**这正是我们的核心竞争力。**
2. **无诊断遥测**：没有类似我们的 dump server 机制。
3. **Popup.tsx 巨石文件**：1597行全在一个 React 组件里，可维护性差。

## 三、战略建议

### 方案对比

| 维度 | 🔨 自建插件 | 🤝 向 Voyager 提交 (Pull Request) |
|------|-----------|------|
| **上手速度** | 快（已有基础） | 慢（需学 TS+React+Vite 工具链） |
| **功能独立性** | ✅ 完全自主 | ❌ 受 maintainer 代码审查约束 |
| **用户触达** | 需自行推广 | ✅ 直接获得现有用户群 |
| **维护负担** | 你承担全部 | ✅ 社区共建 |
| **你的核心价值** | 只属于你 | 贡献给开源社区 |

### 🎯 我的推荐：**混合策略**

1. **短期：继续自建插件**
   - 你的 LaTeX 修复引擎（拆→清理→KaTeX→重包装）是 Voyager **完全没有** 的能力。
   - 你的 Markdown Repair Mode 比 Voyager 的 `markdownPatcher` 更完善。
   - 先把自己的插件打磨到完美，作为"概念验证"。

2. **中期：向 Voyager 提交 PR (Pull Request)**
   - PR 就是"贡献代码"的标准方式。你 Fork 他的仓库，在你的副本里加入你的改进，然后向原作者发 PR。
   - **最适合提交的功能**：LaTeX Fixer 引擎 + KaTeX 重渲染。这是 Voyager 最薄弱的环节。
   - 如果被接受，你的名字会永久出现在贡献者名单中。

3. **长期：从 Voyager 吸收高价值功能到自建插件**
   - **公式复制** — 已研究完毕，可直接集成
   - **引用回复** — 需要研究 DOM 结构
   - **Mermaid 渲染** — 需引入 mermaid 库
   - **提示词管理器** — 独立于 Gemini DOM 的纯 UI 功能

> [!IMPORTANT]
> Voyager 是 GPL v3 许可证，意味着任何基于其代码的衍生品也必须开源并使用相同许可证。如果你直接复制代码到闭源插件，会有法律风险。但如果你是"参考思路后独立实现"，则完全合法。

## 四、"我能装上任意功能吗？"

**技术上：是的，我有这个能力。** 但有实际约束：

| 功能 | 可行性 | 备注 |
|------|--------|------|
| 公式复制 | ✅ 可直接做 | 已完成技术研究 |
| Mermaid 渲染 | ✅ 可做 | 需引入 ~500KB 的 mermaid 库 |
| 引用回复 | ✅ 可做 | 需 DOM 操作 |
| 布局控制 (宽度) | ✅ 简单 | 纯 CSS 注入 |
| 水印移除 | ⚠️ 有条件 | 需 fetch 拦截器，CSP 限制 |
| Google Drive 同步 | ⚠️ 复杂 | 需 OAuth2 配置 + Google API 控制台 |
| 文件夹树 | ⚠️ 极复杂 | Voyager 最复杂的功能，15个文件 |
| 账户隔离 | ⚠️ 极复杂 | 需深度理解 Gemini 路由系统 |
